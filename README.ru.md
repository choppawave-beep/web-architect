# 🏗️ Web Architect — Навык для ИИ-агентов

> **Одна команда. Четыре агента. Никакого шаблонного вывода.**

Навык-оркестратор из 4 агентов для ИИ-помощников, выполняющий полный конвейер **от дизайна до кода** для любого веб-проекта. Одна команда генерирует уникальную дизайн-систему, планирует архитектуру, пишет код и проверяет качество — всё с жёсткими ограничениями, предотвращающими типичный «ИИ-сгенерированный» вид.

Работает с **Claude Code** · **Cursor** · **Amp** · **Codex CLI** · **Gemini CLI** · **GitHub Copilot** · **Windsurf** · и [15+ другими агентами](https://skills.sh)

```bash
npx skills add choppawave-beep/web-architect
```

```
/wa:design homepage with hero, pricing table, and testimonials
```

---

## 😤 Проблема

Каждый сайт, сгенерированный ИИ, выглядит одинаково: шрифт Inter, сине-фиолетовый градиент, центрированная сетка карточек, шаблонные отступы. Навыки Claude Code работают изолированно — вы вручную связываете навыки дизайна, архитектуры и ревью для каждого проекта. Качество непредсказуемо. Контекстное окно раздувается при загрузке нескольких навыков.

## ✨ Решение

`/wa:design` управляет **4 специализированными субагентами** через полный конвейер:

```
🎨 Дизайнер → 🚦 Утверждение дизайна (ваше одобрение) → 📐 Архитектор → 🔨 Строитель → 🔍 Ревьюер → 🔄 Цикл исправлений
```

Каждый агент работает в **чистом контексте** (~5% накладных расходов оркестратора). Данные передаются через файлы на диске. Система **учится на вашей обратной связи** со временем.

## 🧬 Чем он отличается

### 🎯 Генерация с ограничениями

Вместо «сделай современно» Дизайнер применяет **4 уровня ограничений**:

| Уровень | Что делает | Пример ограничения |
|---------|-----------|-------------------|
| 💡 **Концепция** | Выбирает дизайн-метафору из назначения проекта | Должна отличаться от 3 последних проектов того же типа |
| 🎨 **Цвет** | Алгоритм HSL-гармонии (комплементарная/аналоговая/триадная/расщеплённо-комплементарная) | WCAG 4.5:1 для каждой пары текст/фон. Нет чистых #000/#fff |
| ✏️ **Типографика** | Проверенные категории шрифтовых пар | Только Google Fonts. **Запрещены: Inter, Roboto, Arial** |
| 🎬 **Анимация** | Система токенов тайминга с диапазонами | По умолчанию только CSS. `prefers-reduced-motion` обязателен |

### 📊 Оценка качества по 5 измерениям

Ревьюер оценивает каждую сборку по **шкале 0-100 с весами**:

| Измерение | Вес | Что проверяется |
|-----------|-----|----------------|
| 🎨 Соответствие дизайну | 30% | Использование токенов, нет захардкоженных цветов, отражение концепции |
| ♿ Доступность | 25% | Контраст WCAG, состояния фокуса, ARIA, навигация с клавиатуры |
| ⚡ Производительность | 20% | Серверные компоненты, разделение кода, оптимизация изображений |
| 🧹 Качество кода | 15% | TypeScript (без `any`), компоненты до 200 строк |
| 📐 Архитектура | 10% | Соответствие плану, RTL/i18n |

**Оценка < 80?** Строитель автоматически исправляет критические проблемы (максимум 2 раунда). 🔄

### 🗂️ 8 профилей проектов

Каждый профиль определяет паттерны компоновки, приоритеты компонентов, целевые показатели производительности и акценты дизайна:

| Профиль | Подходит для |
|---------|-------------|
| 🚀 `landing` | Маркетинговые страницы, одностраничники, лендинги |
| 💼 `saas` | SaaS-маркетинг + оболочка приложения |
| 📊 `dashboard` | Админ-панели, аналитика, метрики |
| 🛒 `ecommerce` | Магазины, каталоги товаров, оформление заказов |
| 📇 `crm` | Управление контактами, воронки сделок |
| 🖼️ `portfolio` | Креативные витрины, галереи |
| 📝 `blog` | Контентные сайты, статьи, редакционные издания |
| 🖥️ `desktop` | Electron-приложения, инструменты с нативным интерфейсом |

### 🧠 Самообучение

После каждого запуска система извлекает **что сработало / что не удалось / что изменить**. Пользовательские правки на этапе утверждения дизайна (изменение цвета, выбор другого шрифта) получают **наивысший приоритет**. Выводы сохраняются для каждого проекта и загружаются в будущие запуски — система становится лучше с каждым использованием.

## 💡 Использование

### Новый проект

```bash
# 1. Установите тип проекта
/wa:profile landing

# 2. Запустите полный конвейер
/wa:design homepage with hero section, feature grid, pricing table, and testimonials
```

Конвейер выполнит:
1. 🎨 Генерацию уникальной дизайн-системы и **паузу для вашего одобрения** (палитра, шрифты, концепция)
2. 📐 Планирование архитектуры компонентов (структура файлов, потоки данных, порядок сборки)
3. 🔨 Сборку всех компонентов с использованием дизайн-токенов
4. 🔍 Оценку сборки и автоисправление проблем, если оценка < 80

### Существующий проект — добавить страницу

```bash
# Использует вашу существующую design-system/MASTER.md
/wa:build dashboard analytics page with charts and stat cards
```

### Существующий проект — аудит качества

```bash
# Работает с любым кодом, дизайн-система не требуется
/wa:review
```

### 📄 Выходные артефакты

| Файл | Назначение |
|------|-----------|
| `design-system/MASTER.md` | 🎨 CSS custom properties, концепция, шрифты, токены анимации |
| `web-architect-plan.md` | 📐 Дерево компонентов, спецификации файлов, порядок сборки |

| `web-architect-review.md` | 🔍 Оценка по 5 измерениям с выводами |

## 🏛️ Архитектура

```
┌──────────────────────────────┐
│   /wa:design command          │  ← 🧠 Оркестратор (~5% контекста)
│   (commands/wa/design.md)     │
└───────────┬──────────────────┘
            │ последовательный запуск агентов
     ┌──────┼──────────┬──────────────┐
     ▼      ▼          ▼              ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│🎨 Дизн │→│📐 Арх  │→│🔨 Сбор │→│🔍 Ревь │  ← Чистый контекст для каждого
└────┬───┘ └────┬───┘ └────┬───┘ └────┬───┘
     │          │          │          │
     ▼          ▼          ▼          ▼
  MASTER.md   plan.md   код файлы  review.md  ← Файлы на диске
```

**Нулевое раздувание контекста**: Оркестратор никогда не читает полное содержимое файлов — только метаданные (оценка, количество файлов). Каждый субагент получает 100% чистый контекст только с необходимыми файлами.

## 🔧 Расширение

**➕ Добавить профиль проекта** — Дополните `references/project-profiles.md`. Изменения кода не требуются.

**➕ Добавить измерение качества** — Отредактируйте `references/quality-standards.md`. Обновите веса (сумма должна быть 100%).

**➕ Настроить ограничения** — Отредактируйте `references/constrained-generation.md`. Добавьте шрифтовые пары, типы гармонии или паттерны анимации.
